Motion tokens

Design-direction agnostic

All animation durations and easing functions are defined as tokens. These do not change when the visual design direction is swapped — motion is structural, not cosmetic.

instant
0ms
State flags, boolean toggles. No visual delay.
quick
120ms
Hover bg, border colour changes, pill state shifts.
standard
200ms
Default for most UI transitions. Drawers, dropdowns, focus rings.
deliberate
350ms
Progress indicators, step advancement, confirmation banners.
exit / collapse
400–500ms
Row fade-and-collapse after send. Gives user time to register completion before the row disappears.
colour token note
Pending direction lock
The accent colour used in toasts, momentum prompts, and milestone icons is currently Electric Dreams cyan. It will update to the primary accent of the chosen direction when locked.

Easing functions

Enter (spring)
cubic-bezier(0.22, 0.68, 0, 1.2)
Exit
ease-in
Default
ease

The spring easing (enter) gives toasts and momentum prompts a slight overshoot — just enough to feel alive, not playful enough to feel gamified.

Microcopy system

Locked — Phase 2.12

All short-form UI text — buttons, confirmations, empty states, errors, guidance — lives in version-controlled JSON files. No CMS. No database. Copy is product behaviour.

Voice rules

Avoid
  • configure
  • execute
  • initiate
  • parameter
  • Error 400
  • Template successfully generated
  • You are required to…
Prefer
  • write
  • send
  • add
  • follow up
  • Here's what happened
  • We've prepared a message
  • Add at least one contact

File structure

src/config/microcopy/
├── activation-loop.json
├── message-editor.json
├── empty-states.json
├── success-states.json
├── error-messages.json
├── reassurance.json
├── common.json
└── index.ts

One key = one string. Separate keys for singular/plural variants (cta_add_prospect / cta_add_prospects). Updated via Git PR, not database. Error events tracked in analytics on show.

Copy update policy: Microcopy is treated as product behaviour, not marketing content. Any change requires Git commit → PR → review → deploy. This ensures staging verification and version history.

Momentum engine

Locked — Phase 2.13

After every meaningful action, the product answers the question: "What should I do next?" Momentum prompts are short, contextual, and optional. They never pressure. They never block.

Placement rules

Prompts appear beneath completion confirmations, within onboarding steps, in empty states, and after key product actions.

They never appear in modals. They never block the current workflow. They auto-expire after 10 seconds or dismiss on the user's next action.

The prompt uses a left-border accent (the primary colour of the active design direction). border-radius: 0 on left side — no rounded corner on a single border.

Milestone recognition

Important product moments feel rewarding. Subtle but not invisible. Professional but not cold. No confetti. No sound. The restraint is the design.

What milestones deliberately avoid

Confetti animations · Streak pressure · Gamification points · Leaderboards · Forced engagement loops · Blocking overlays · Sound effects

The product should feel like a professional tool that celebrates real work, not a game that celebrates engagement.
