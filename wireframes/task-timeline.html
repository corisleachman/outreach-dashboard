Notes — chip states
No tasks: no chip. Clean row, no noise.
Open tasks: amber chip with count. Single chip, always visible.
Open + done: two chips — amber for open count, muted grey for done count. Done chip is secondary, never dominant.
Overdue: red chip replaces amber. Row gets a faint red left-border accent. Overdue always surfaces above other states.
All done, none open: grey muted chip with done count. History is visible but not urgent — can be omitted if it adds too much noise.
Clicking either chip opens the drawer directly to the Tasks tab — no intermediate step.

Notes — completion prompt
Prompt appears in the drawer inline below the completed task — not a modal, not a toast. It's part of the flow, not an interruption.
"Set a follow-up" opens the task creation modal with the same prospect pre-filled and due date defaulting to 3 days (most likely follow-up cadence).
"Not now" dismisses the prompt. It doesn't reappear — the user can always set a new task manually via the Add task button.
Prompt auto-dismisses if the user closes the drawer without interacting with it.

Notes — inline edit
Edit expands inline below the task header — no modal, no page change. The timeline stays visible while editing.

Notes — full history
Open tasks always appear at the top of the timeline — newest due date first. Done tasks follow below in reverse chronological order.
The timeline is a record of intent and action — not just a to-do list. Done tasks are preserved permanently (no bulk clear or archive).
The connecting line between items is intentional — it makes the sequence readable as a story: "I did this, then this, now I need to do this."
The drawer tabs (Details / Tasks) let users switch between the contact info view and the task timeline without closing and reopening.
